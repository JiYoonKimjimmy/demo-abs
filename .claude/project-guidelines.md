# ABS 프로젝트 작업 가이드라인

## Git 커밋 규칙

### 자동 커밋 금지
작업 완료 후 자동으로 Git 커밋을 진행하지 않습니다.

**원칙**:
- 모든 커밋은 사용자의 **명시적 요청**이 있을 때만 실행
- 사용자가 "깃 커밋 진행하자" 또는 `/commit` 커맨드를 입력할 때만 커밋 진행
- 작업 완료 시 커밋 여부를 묻지 않고, 사용자의 다음 지시를 기다림

**예시**:

❌ **잘못된 패턴** (자동 커밋):
```
Assistant: 문서 작성이 완료되었습니다!
[자동으로 git add와 git commit 실행]
```

✅ **올바른 패턴** (명시적 요청 대기):
```
Assistant: 문서 작성이 완료되었습니다!
[커밋 없이 대기]

User: 깃 커밋 진행하자
Assistant: [git status 확인 후 커밋 진행]
```

### 커밋 실행 트리거

다음 중 하나의 요청이 있을 때만 커밋을 실행합니다:

1. **명시적 요청**: "깃 커밋 진행하자", "커밋해줘", "git commit 해줘"
2. **슬래시 커맨드**: `/commit`

### 커밋 프로세스

커밋 요청이 있을 때:

1. `git status` 실행하여 변경사항 확인
2. `git diff` 실행하여 변경 내용 상세 확인
3. 변경 내용 분석하여 적절한 커밋 메시지 작성
4. `git add`와 `git commit` 실행
5. `git status`로 커밋 완료 확인

### 커밋 제외 파일

다음 파일들은 절대 커밋하지 않습니다:
- `.DS_Store` (macOS 시스템 파일)
- `.env`, `.env.local` (환경변수 파일)
- `node_modules/`, `vendor/` (의존성 디렉토리)
- 기타 `.gitignore`에 정의된 파일

## 작업 완료 보고

### 작업 완료 시 행동

작업 완료 후:
1. **완료된 작업 요약**: 어떤 작업을 완료했는지 명확히 보고
2. **생성/수정된 파일 목록**: 변경된 파일 나열
3. **다음 단계 제안** (선택사항): 필요시 다음 작업 제안
4. **커밋 대기**: 사용자의 명시적 요청 대기

**예시**:
```
완료되었습니다! 01-architecture/ 아키텍처 설계 문서 작업이 완료되었습니다.

## 작성 완료된 문서
- docs/01-architecture/01-architecture-overview.md
- docs/01-architecture/02-project-structure.md
- docs/01-architecture/03-layer-design.md
- docs/01-architecture/04-dependency-injection.md

다음으로 02-domain/ 도메인 모델 설계 문서를 작성하시겠습니까?
```

## 체크리스트 관리

### 체크리스트 업데이트 규칙

문서 작업 완료 시:
1. 사용자가 별도로 요청하지 않으면 체크리스트를 자동으로 업데이트하지 않음
2. 사용자가 "체크리스트 업데이트해줘" 또는 유사한 요청을 할 때만 업데이트
3. 체크리스트 업데이트 후에도 자동으로 커밋하지 않고, 별도의 커밋 요청 대기

## TodoWrite 도구 사용

### 사용 시점
- 복잡한 멀티 스텝 작업 시작 시
- 사용자가 명시적으로 todo 관리를 요청할 때

### 사용 금지 시점
- 단순한 파일 수정 작업
- 이미 완료된 작업 보고 시
- Git 커밋 프로세스

---

## 요약

**핵심 원칙**:
1. ✅ 작업 완료 → 사용자에게 보고 → 대기
2. ❌ 작업 완료 → 자동 커밋 (금지)
3. ✅ 사용자 요청("깃 커밋 진행하자") → 커밋 실행

**슬래시 커맨드**:
- `/commit`: Git 커밋 실행
